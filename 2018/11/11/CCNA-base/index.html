#CCNA# 路由基础

2018.11.11 D-Veda 计算机  热度 ℃

路由基础

  • 1个网段就是1条路由
  • 整个路由部分就是围绕如何帮助路由器收集所有路由条目
  • 路由器必须拥有网络内的所有路由条目, 才能转发各网段数据包

路由

  • 作用: 数据从源设备成功到达目标设备的路径总和

路由类型

  • 本地路由: 路由器直连网段
  • 外地路由: 非路由器直连网段

获悉外地路由方式有两种

静态路由: 管理员手动配置未知网段
动态路由: 路由器自动学习未知网段

路由表

  • 作用: 装载路由的表项

路由表实例

路由表中路由条目类型

  • 直连路由: 本地直连接口–C
  • 静态路由: 通过手动配置的非本地网段–S
  • RIP路由: 通过动态RIP学习到的非本地网段–R
  • EIGRP路由: 通过动态EIGRP学习到的非本地网段–D
  • BGP路由: 通过动态BGP学习到的非本地网段–B
  • OSPF路由: 通过动态OSPF学习到的非本地网段–O
  • ISIS路由: 通过动态ISIS学习到的非本地网段–I

路由器

  • 作用: 负责学习路由, 生成路由表, 依据路由表转发数据包
  • 路由工作在网络层

路由器工作原理

  • 根据路由表转发数据
  • 对已知数据包 -> 转发
  • 对未知数据包 -> 丢弃

路由器硬件组成

路由器启动过程

  • (1)开机自检(power-on self, POST)
  • (2)如果(1)正常, IOS存在, 将从闪存FLASH中查找和加载IOS到随机存储器RAM中(但2500系列不加载RAM中, 直接从闪存中运行)
  • (3)如果(1)和(2)正常, 接下来它将在非易失存储器NVRAM中查找启动配置文件startup-config, 假如没有找到任何启动配置文件, router将进入到SETUP模式
  • (4)如果在非易失存储器NVRAM中查找到启动配置文件startup-config, 将按照启动配置文件startup-config启动路由器

Cisco路由器CLI配置命令

Router> //用户模式
Router# //特权模式: 检查设备的运行状态
Router(config)# //全局配置模式: 可以配置命令
Router(config-if)# //接口配置模式
Router(config-router)# //路由配置模式
Router(config-line)# //线路配置模式

CLI配置模式

  • 从上到下输入命令进入不同的模式

  • exit 是回退上一级

  • end 回到特权模式

  • cisco的命令的特点

    1. cisco的命令可以简写
    2. cisco的命令可以补全(按Tab键)
